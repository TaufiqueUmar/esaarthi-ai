📦 Component Reference

All files in src/voice-agent/

🌐 VoiceFlowAgent.jsx

  • Global React Context Provider wrapping entire app
  • Orchestrates speak → listen → process loop
  • Page-aware: speaks prompt on every route change
  • Exports useVoiceFlowCtx for pages
  • DOM injection: fillInput, fillTextarea, fillSelect, clickButton
  • Handles: Bill, Complaint, OTP, Status, Navigation intents

🎤 useSarvamSTT.js

  • navigator.mediaDevices.getUserMedia (16kHz mono)
  • AudioContext ScriptProcessor (4096 buffer)
  • RMS silence detection threshold: 0.006
  • Auto-stop: 2.5s silence OR 25s hard cap
  • Float32 → 16-bit PCM WAV encoder (44 byte header)
  • POST to Sarvam saaras:v3 REST API

🔊 useSarvamTTS.js

  • POST text to Sarvam bulbul:v3 REST API
  • Speaker: shubh (Hindi male voice)
  • Text chunked to ≤490 chars at sentence boundaries (।/.)
  • 3 auto-retries on network failure
  • base64 WAV → atob → decodeAudioData → BufferSource
  • 450ms grace period before re-arming STT after TTS

🧠 intentEngine.js

  • Pure rule-based NLU — no LLM, no API
  • Special intents: greet, home, yes, no, repeat, help
  • Flow intents scored by keyword weight (multi-word = higher score)
  • Cross-language: matches Hindi + English + Hinglish
  • Department detection: gas / electricity / municipal
  • Confidence: 0–1 (score / 10, capped)

🔄 dialogManager.js

  • State machine: greeting → flow_active → done
  • process(userInput) is the main entry point
  • Step types: input, choice, confirm, navigate, done
  • Template filling: _AMOUNT_, _MONTH_, _STATUS_, _DATE_
  • failCount tracks retries; auto-resets on home/advance
  • Used by dialog flows, not the main VoiceFlowAgent directly

🗺️ useVoiceNav.js

  • Lightweight navigation-only hook (Language.jsx)
  • resolveRoute() maps transcript → route string
  • Supports same Hindi/English/Hinglish keyword set
  • Speaks confirmation, navigates, then goes idle
  • Does NOT do form-filling (that's VoiceFlowAgent's job)
